# Icon Design

The two decisions every icon needs:

1. **Which icon** ŌĆö the right symbol for the concept
2. **Which animate mode** ŌĆö the motion that matches the interaction

## Choosing `animate` Mode

| Mode | When to use | Feel |
|------|-------------|------|
| `"hover"` | Primary action buttons, navigation links | Reactive ŌĆö responds to intent |
| `"click"` | Confirmation actions, toggles, destructive ops | Decisive ŌĆö responds to commitment |
| `"loop"` | Loading states, status indicators | Continuous ŌĆö never stops |
| `"once"` | Success states, completion badges | Definitive ŌĆö plays once |
| `false` | Decorative icons, dense icon grids | Static ŌĆö no motion |

**Rule of thumb**: Use `"hover"` for navigation/browsing icons, `"click"` for action icons.

## Sizing

| Context | `size` prop | Tailwind wrapper |
|---------|-------------|-----------------|
| Inline with text (16px body) | `size={14}` | `text-sm` |
| Inline with text (default) | `size={18}` | `text-base` |
| Button icons | `size={20}` | ŌĆö |
| Feature cards | `size={32}` | ŌĆö |
| Hero / marketing | `size={48}` | ŌĆö |
| Large decorative | `size={64}` | ŌĆö |

The default is `size={24}`. Motion amplitudes in the animation defs are tuned for 24px ŌĆö larger icons feel more stable at the same keyframe values because absolute pixel displacement scales down relative to icon surface area.

## Source Selection

liveicons exposes five icon sets. All share the same animation system.

| Import | Style | Best for |
|--------|-------|----------|
| `liveicons/react` | Lucide (stroke) | Default ŌĆö general use |
| `liveicons/react/tabler` | Tabler (stroke) | Denser, more technical feel |
| `liveicons/react/tabler-filled` | Tabler (fill) | Solid style, CTAs |
| `liveicons/react/heroicons` | Heroicons (stroke) | Tailwind / minimal projects |
| `liveicons/react/heroicons-solid` | Heroicons (fill) | Solid style, Tailwind CTAs |

**Consistency rule**: never mix stroke and fill icons in the same section.

## Usage Examples

### Notification button

```tsx
import { BellIcon } from 'liveicons/react'

<button>
  <BellIcon animate="hover" size={20} />
  Notifications
</button>
```

### Submit / send action

```tsx
import { SendIcon } from 'liveicons/react'

<button onClick={handleSubmit}>
  <SendIcon animate="click" size={18} />
  Send
</button>
```

### Loading state

```tsx
import { LoaderIcon } from 'liveicons/react'

{isLoading && <LoaderIcon animate="loop" size={20} />}
```

### Success feedback

```tsx
import { CheckIcon } from 'liveicons/react'

{success && <CheckIcon animate="once" size={20} />}
```

### Dark/light toggle

```tsx
import { SunIcon, MoonIcon } from 'liveicons/react'

<button onClick={toggleTheme}>
  {isDark ? <SunIcon animate="click" size={20} /> : <MoonIcon animate="click" size={20} />}
</button>
```

### Imperative control (ref)

```tsx
import { useRef } from 'react'
import { BellIcon } from 'liveicons/react'
import type { LiveIconHandle } from 'liveicons/react'

const bellRef = useRef<LiveIconHandle>(null)

// Trigger programmatically (e.g. on new notification):
bellRef.current?.startAnimation()

<BellIcon ref={bellRef} size={20} />
```

## Speed

```tsx
<BellIcon animate="hover" speed="fast" />   // 0.25s
<BellIcon animate="hover" speed="normal" />  // 0.6s (default)
<BellIcon animate="hover" speed="slow" />    // 1.2s
<BellIcon animate="hover" speed={0.4} />     // exact seconds
```

Use `speed="fast"` for click/tap feedback icons. Use `speed="slow"` for attention-drawing notifications.

## Common Mistakes

| Mistake | Fix |
|---------|-----|
| Using `"loop"` on action icons (heart, check) | Use `"click"` ŌĆö looping heartbeats are anxious, not delightful |
| Using `"hover"` on destructive actions (trash) | `"hover"` warns; but pair with a confirm pattern, not direct delete |
| Mixing Lucide + Heroicons in one section | Pick one style per component/section |
| Setting `animate={false}` on every icon in a dense list | That's correct ŌĆö don't animate icon-heavy tables or nav menus |
| Using `"once"` for a loading spinner | Use `"loop"` ŌĆö once plays the animation one time and stops |
